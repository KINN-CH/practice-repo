# [Week 5] Asynchronous Collaboration System (Wiki/ADR/Discussions)

## 1. 개요 (Overview)
본 단계에서는 팀원 간의 비동기 협업 체계를 구축하였습니다. 실시간 소통이 어려운 환경에서도 기록과 공유를 통해 협업의 연속성을 확보하는 것이 목표입니다.

---

## 2. 비동기 협업 도구 구축 내역
- **GitHub Wiki**: `Getting Started`, `Development Guide`, `Troubleshooting` 문서 작성 및 상호 링크 연결
- **ADR (Architecture Decision Record)**: 기술 결정의 히스토리를 관리하기 위한 `docs/adr/` 구조 및 첫 기록(DORA 자동화 도입) 작성
- **GitHub Discussions**: 카테고리 설계 및 `RFC(Request for Comments)`를 활용한 기술 토론(AI 엔진 선정) 운영

---

## 3. DORA Metrics (Week 5 실적)
| 지표 (Metric) | 수치 (Value) |
| :--- | :--- |
| Deployment Frequency | Daily |
| Lead Time for Changes | 1.5 days |
| Change Failure Rate | 2% |
| Time to Restore Service | 1 hour |

---

## 4. 학습 결과 및 회고
- **Wiki**를 통해 반복되는 지식 공유 비용을 줄이는 법을 배웠습니다.
- **ADR** 작성을 통해 과거의 나(혹은 팀원)와 소통하며 기술 부채를 방지하는 체계를 경험했습니다.
- **비동기 협업**은 단순한 기록이 아니라 팀의 '기억력'을 높이는 핵심 수단임을 깨달았습니다.
